## 公网转发工具：FRP

**为什么需要公网转发？**

出于对服务器等设备的保护，现代的计算机网络发展出了一套有效防御外部恶意攻击的手段，采用诸如防火墙、网络地址转换NAT（Network Address Translation）（单独划分出一些只在内部网络起作用的ip地址）等手段，划分出了“**内部**”和“**外部**”的概念。这样的“一刀切”虽然很好地阻隔了外部攻击，但也反过来阻止了一些内网中服务、请求向外网的直接传播。

从云服务商租来的服务器总是处于这种阻隔保护之内的。因此，如果想让服务器真正提供服务，就应打通内外网之间的链接。

公网转发就像是给处于内网的服务器搭了一座桥，让外网的玩家也能访问到你位于内网的服务器。FRP 是一个轻量级的内网穿透工具，能把内网服务映射到公网。

FRP只需要配置一个 `frpc.toml` 文件，就能快速实现转发。它还支持多种可靠传输协议（如 TCP、HTTP），性能可靠。

**举个例子：**
假如你的服务器内网 IP 是 `192.168.1.2`，端口是 `11451`（在MC中可见的，发出多人游戏服务的端口，即内端口），你希望外网玩家通过 `1145` 端口访问（对外暴露的、外部玩家用来访问的端口，即外端口）。通过 FRP 配置后，外网的请求会被转发到你的服务器上。

详细配置请参考：[FRP 配置指南](./docs/server_arch/env_cfg/frp/FRP-doc.md)。
